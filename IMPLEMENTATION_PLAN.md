# QuickAgeCalc implementation plan

Updated: 2026-07-18

## Goals

- Expand search coverage beyond nearly identical birth-year pages.
- Build useful calculators around real questions seen on Quora and Pinterest.
- Improve internal linking, annual maintenance, trust and crawl paths.
- Release changes in small, verifiable batches.

## Week 1 — Technical foundation and first search-intent pages

### Day 1
- Add `/age-questions/` content hub.
- Add `/age-in-any-year/` calculator.
- Add `/what-year-was-i-born/` calculator.
- Add `/what-generation-am-i/` calculator.
- Add the 2026 birth-year page.
- Update global navigation, sitemap and deployment workflow.
- Prevent duplicate global footer insertion.
- Make current-year corrections during deployment.

### Day 2
- Add prominent links from the homepage to the new calculators.
- Add links from birth-year pages to the question hub and related calculators.
- Verify canonicals, titles, descriptions and mobile layout.

### Day 3
- Improve the About page with calculation methodology, privacy and editorial/testing process.
- Add a visible “last reviewed” pattern for calculator pages.
- Add Organization/WebSite structured data where appropriate.

### Day 4
- Create the first high-value Q&A pages based on observed demand:
  - How old was I in a specific year?
  - How old will I be in 2030?
  - When will I turn 40, 50 or 65?
  - Millennial or Gen Z?
  - Gen Z or Gen Alpha?

### Day 5
- Create decade hubs for the 1970s, 1980s, 1990s and 2000s.
- Strengthen links between decade hubs, year pages and calculators.

### Day 6
- Review Search Console query/page data supplied by the owner.
- Improve titles and snippets for pages with impressions but low CTR.
- Check for duplicated or weak pages before adding more programmatic URLs.

### Day 7
- Verify deployed URLs, sitemap discovery and workflow health.
- Publish a weekly implementation report.
- Set priorities for Week 2 from real impressions and outbound-click data.

## Week 2 — Q&A cluster and internal linking

- Publish 10–15 substantial question pages, not thin one-sentence answers.
- Each page must include a direct answer, interactive calculator or table, examples and contextual internal links.
- Add question categories: Past Age, Future Age, Birth Year, Milestones and Generations.

## Week 3 — Content quality and differentiation

- Add milestone tables and age timelines to the strongest birth-year pages.
- Add shareable result text and copy buttons where useful.
- Improve source notes on dog-age, pregnancy and retirement pages.

## Week 4 — Optimization from data

- Compare impressions, CTR, position and outbound traffic by landing page.
- Expand only clusters showing demand.
- Consolidate or noindex pages that remain duplicative and provide no distinct value.

## Release rules

- Do not mass-publish hundreds of near-identical Q&A pages.
- Every new URL must answer a distinct search intent.
- Keep the primary focus on age, dates, birth years, future/past age and generations.
- Do not claim deployment success until the workflow status is verified.
